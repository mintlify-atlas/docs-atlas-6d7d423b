## What is Pupload?

Pupload is an open source system for handling user uploads, applying media transformations, and storing results exactly where you want them. It supports writing custom tasks, combining them into flows, and managing the lifetime of results and intermediaries all in one place.

Most workflow engines are built for generic task orchestration. **Pupload is built specifically for files.**

## Core concepts

<AccordionGroup>
  <Accordion title="Flows" icon="water">
    YAML pipelines that define your processing workflow. They contain nodes (steps), edges (connections), datawells (input/output), and stores (storage backends).
  </Accordion>
  
  <Accordion title="Nodes" icon="cube">
    Run in Docker containers. Each node gets presigned URLs for inputs/outputs. On completion, it uploads results and triggers downstream nodes.
  </Accordion>
  
  <Accordion title="Resource tiers" icon="microchip">
    Workers subscribe to resource queues:
    - `c-small/medium/large` - CPU
    - `m-small/medium/large` - Memory  
    - `g-small/medium/large` - GPU (NVIDIA, AMD, Intel, Apple)
    - `gn-*/ga-*/gi-*` - Vendor-specific
  </Accordion>
  
  <Accordion title="Stores" icon="database">
    Storage backends for your data. Supports S3-compatible storage with configurable buckets and endpoints.
  </Accordion>
</AccordionGroup>

## Common use cases

<CodeGroup>

```yaml Image/Video Processing
upload → thumbnail → watermark → multiple resolutions → CDN
```

```yaml Document Pipeline
PDF upload → extract text → OCR scanned pages → index → store
```

```yaml ML Inference
upload → preprocess → GPU inference → postprocess → results
```

```yaml Archive Processing
zip upload → extract → virus scan → process each file → store
```

</CodeGroup>

<Tip>
Mix CPU and GPU nodes. Scale GPU workers separately. Run expensive steps on dedicated hardware.
</Tip>

## Why Pupload vs rolling your own?

You don't have to build:
- Presigned URL handling
- Container orchestration
- Retry logic
- Resource scheduling
- Distributed state management

<Note>
Pupload requires **Go 1.25+** and **Docker**. Redis is optional for standalone server deployments.
</Note>
